# 上置信界算法和汤普森采样算法
## 上置信界算法
上置信界算法是一种经典的基于不确定性的策略算法，其核心为**霍夫丁不等式**
*霍夫丁不等式：*
对n个取值范围为[0,1]的，独立同分布的随机变量$X_1,X_2,...,X_n$，其经验期望为$\bar{x_n} = \frac{1}{n}\sum_{j=1}{n}X_j$
有
$$P(E[X] - \bar{x_t} \geq u) \leq e^{-2nu^2}$$
- 样本越多($n\uparrow$)，实际概率分布期望与采样概率均值的偏差大于不确定性的概率就越小
- 不确定性大($u\uparrow$)，容许更大的偏差，则偏差大于不确定性的概率也小
公式也可以变形为：
$$P(E[X] \geq \bar{x_t} + u) \leq e^{-2nu^2}$$
可以理解成使用采样平均和不确定性对实际概率分布期望进行逼近


对于上置信界算法，使用第二种理解方式，当设定的概率p很小时，$E[X] \geq \bar{x_t} + u$就有大概率（1-p）成立，此时认为$\bar{x_t} + u$是概率分布均值的上界，记作上置信界。我们还可以设定一个系数来控制不确定性所占权重，即$\bar{x_t} + cu$

在MAB问题中，上置信界算法选择$\bar{x_t} + u$大的动作执行。
不妨给定概率函数：
$$p = e^{-2n\hat{u}^2}$$
不难解出
$$\hat{u} = \sqrt{\frac{ln(e)}{-2n}}$$
此时$\hat{u}$即为不确定性的上限。
不妨令$p = \frac{1}{t}$
则可以求出不确定性上限，并与采样均值加和作为期望奖励。
随着时间增加，允许的误差概率p越来越小，采样次数n越来越大，此时不确定性上限也随之减小，采样均值也越来越接近真实期望。
在实际的问题中，由于拉杆被选择的次数可能为0，所以使用n+1作为拉杆被拉动的次数。
开始交互时，每次选择奖励的上置信界最大的动作，反复执行更新。

## 汤普森采样算法
假设拉动每根拉杆得到的奖励服从某种概率分布，则可以根据该拉杆概率分布的期望来进行动作的选择。

由于计算所有拉杆的期望奖励的难度较高，汤普森采样算法使用采样的方式，根据当前每个动作$a$的奖励概率分布进行一轮采样，得到一组各拉杆的奖励样本，选择其中奖励最大的动作。

**汤普森采样是一种计算所有拉杆的最高奖励概率的蒙特卡洛方法**

### 动作a的奖励概率分布和更新
在实际情况中，通常使用**Beta分布**对当前每个动作的概率分布进行建模。
*若某根拉杆被选择了$k$次，其中$m_1$次奖励为1，$m_2$次奖励为0，则该拉杆的奖励分布服从$(m_1+1, m_2+1)的Beta分布$*
> Beta分布：
> 由两个正参数$\alpha > 0, \beta > 0$确定，定义在[0,1]区间上的连续概率分布
> 当$\alpha > \beta > 1$时，分布右偏，峰值靠近1。
> 当$1 < \alpha < \beta$时，分布左偏，峰值靠近0。
> 当$\alpha = \beta$时，分布对称。
> 当$\alpha = \beta = 1$时，分布为一条水平直线。
> 当$\alpha, \beta$二者有一小于1时，分布变为凹函数；反之，分布为凸函数。


在Python中，可以通过`np.random.beta(para_alpha, para_beta)`来构造Beta分布。
在MATLAB中，可以通过`betapdf(x, para_alpha, para_beta)`来构造Beta分布，其中x为分布区间[0,1]。

## 代码实现
### 上置信界算法
```python
class UCB(Solver):
    def __init__(self, bandit, init_prob=1.0, coef):
        super().__init__(bandit)
        self.total_count = 0
        self.estimate = np.array([init_prob]*self.bandit.K)
        self.coef = coef

    def run_one_step(self):
        self.total_count += 1
        ucb = self.estimates + self.coef * np.sqrt(log(self.total_count)/(2*(self.counts + 1)))    # python中ln用log表示，lg用log10表示。
        k = np.argmax(ucb)
        r = self.bandit.step(k)
        self.estimates[k] += 1./(self.estimates[k]+1)*(r-self.estimates[k])
        return k
```

### 汤普森采样算法
```python
class ThompsonSampling(Solver):
    def __init__(self, bandit):
        super().__init__(bandit)
        self._a = np.ones(self.bandit.K)
        self._b = np.ones(self.bandit.K)

    def run_one_step(self):
        samples = np.random.beta(self._a, self._b)
        k = np.argmax(samples)
        r = self.bandit.step(k)

        self._a[k] += r
        self._b[k] += (1-r)
        return k

```
